Работу можно разделить на несколько блоков:

  1. Изучение теории. Результатом будет презентация.
    • В презентацию включаем теорию с краткими примерами;
    • Презентацию сохраняем в формате .pdf;
    • Слайды презентации с темным шрифтом на светлом фоне, читабельные;
    • В задании перечислен минимальный список, который нужно раскрыть. Его можно расширять. Особенно, если у казано "и др."
    • Не переходите на "чужие" вопросы. Расписывайте ваш.
  2. Составление вопросов для самопроверки по теории. Это можно оформить отдельным файлом или добавить в презентацию.
    • Вопросы должны покрывать все ключевые моменты. Рекомендуемое количество 10-20;
    • Вопросы могут быть следующих форматов:
      • С 4+ вариантами ответов (1 правильный);
      • С несколькими ответами (выбрать из списка, в котором не менее 4 вариантов);
      • Ответы формата ДА/НЕТ;
      • Сопоставить 4+ пары;
      • Ввести ответ (для простых ответов);
    • Под вопросами привести правильные ответы.
  3. Практика. Примеры.
    • Не делаем сложных примеров;
    • По возможности берем их из жизни;
    • Примеры делаем на все ключевые моменты.
      Например, для CSS свойства text-align возможны следующие значения: start , end , left , right , center , justify , match-parent , -moz-center , -webkit-center , inherit , initial , revert , revert-layer , unset. Из них основными являются: start , end , left , right , center , justify , match-parent. На каждое из них и приводим пример. Аналогично и для атрибутов html;
    • Примеры должны быть "живыми". То есть в файлах html и css. Таким образом их легко можно будет запустить и попробовать.
  4. Практика. Задания для самостоятельной работы Тут нужно подготовить заготовки для заданий и для каждой описать задачу.
    • Задания не должны быть сложными - минут на 5-15 самостоятельной работы;
    • Рекомендованное количество заданий: 2-3;
    • Тут ответы не нужны, но вы должны знать, как его выполнить без шпаргалок.

Работа с репозиторием:

  1. Ссылка на проект
  2. Ваш объем работы по проекту ~ объему работы по вашим комитам. Так что комиты со своей работой делайте от своего имени.
  3. Над каждым докладом работайте в своей папке и только в своей ветке. Их названия совпадают с номером доклада.
  4. Когда вы хотите отправить работу на проверку, нужно сделать pull request из своей ветки (в которой сделаны все commit и push) в ветку main.
  5. Стили у нас находятся в папке css. Для каждого доклада файл со своим номером. Добавить при необходимости файлы туда можно, но называйте в виде ВашНомер_НомерСтилевогоФайла.css
  6. Там же (в папке css) при необходимости положите и файлы шрифтов. Их не нужно специально переименовывать. Пусть название файла соответствует названию шрифта.
  7. Для остальных ресурсов (картинки, видео, аудио, др.) создайте папку resource в папке с вашим докладом и сложите туда.
  8. Не добавляйте большие по объему ресурсы.

О содержании

  • В блоке HTML (до горизонтальной линии) как минимум:
    Для каждого тега указать
    1. его смысл;
    2. нужно ли его закрывать;
    3. где может быть расположен (если есть нюансы);
    4. какие есть дочерние (из тех, что связаны по смыслу). Например в tr м.б. td и th. А то, что внутри div м.б. span указывать не нужно;
    5. какие есть атрибуты (характерные для рассматриваемого тега) и какие значения они принимают и какое влияние оказывают.
  • В блоке CSS (после горизонтальной линии) как минимум:
    Для каждого стилевого свойства указать
    1. его смысл;
    2. зачастую одно стилевое правило может содержать в себе несколько. О таком коротком способе записи нужно рассказать. Например, magrin, padding, border, font, ...
    3. все значения (кроме inherit) стилевых правил. Возможно, какие-то еще (кроме inherit) можно будет пропустить, но это индивидуально.
    4. для каких тегов может применяться (если есть нюансы).
  • Прочитайте перед написанием примеров статью "Как строить структуру CSS и называть классы".
