Списки, таблицы, ссылки и медиа теги

Теория по тегам

1. Маркированный список (<ul>)

  • Смысл: Создает список с маркерами (точками).
  • Закрывать: Да (</ul>).
  • Где расположен: Внутри body или других блоков.
  • Дочерние теги: Только <li>.
  • Атрибуты: type (вид маркера: disc, circle, square).

2. Нумерованный список (<ol>)

  • Смысл: Создает нумерованный список.
  • Закрывать: Да (</ol>).
  • Где расположен: Внутри body.
  • Дочерние теги: Только <li>.
  • Атрибуты: type (тип счета: 1, A, a, I, i), start (с какого числа начать), reversed (обратный порядок).

3. Элемент списка (<li>)

  • Смысл: Отдельный пункт списка.
  • Закрывать: Да (</li>).
  • Где расположен: Только внутри <ul> или <ol>.
  • Дочерние теги: Любой текст, картинки или вложенные списки.
  • Атрибуты: value (задать конкретный номер пункта, только для ol).

4. Список определений (<dl>, <dt>, <dd>)

  • Смысл: Список терминов и их описаний.
  • Закрывать: Да все три тега.
  • Где расположен: Внутри body.
  • Дочерние теги: Внутри <dl> только <dt> (термин) и <dd> (описание).
  • Атрибуты: Нет специфических.

5. Таблица (<table>)

  • Смысл: Контейнер для таблицы.
  • Закрывать: Да (</table>).
  • Где расположен: Внутри body.
  • Дочерние теги: <caption>, <thead>, <tbody>, <tfoot>, <tr>.
  • Атрибуты: border (рамка, устаревший).

6. Строка таблицы (<tr>)

  • Смысл: Строка таблицы.
  • Закрывать: Да (</tr>).
  • Где расположен: Внутри <table>, <thead>, <tbody> или <tfoot>.
  • Дочерние теги: <td> или <th>.
  • Атрибуты: Нет часто используемых специфических атрибутов.

7. Ячейка (<td>) и Заголовок (<th>)

  • Смысл: <td> - данные, <th> - заголовок столбца/строки.
  • Закрывать: Да.
  • Где расположен: Только внутри <tr>.
  • Дочерние теги: Любой контент.
  • Атрибуты: colspan (объединить столбцы), rowspan (объединить строки), scope (для th).

8. Группы таблицы (<thead>, <tbody>, <tfoot>)

  • Смысл: Шапка, тело и подвал таблицы.
  • Закрывать: Да.
  • Где расположен: Прямо внутри <table>.
  • Дочерние теги: Только <tr>.
  • Атрибуты: Нет специфических.

9. Изображение (<img>)

  • Смысл: Вставка картинки.
  • Закрывать: Нет (одинарный тег).
  • Где расположен: Где угодно, где есть текст.
  • Дочерние теги: Нет.
  • Атрибуты: src (путь к файлу, обязательно), alt (описание), width/height (размеры).

10. Видео (<video>) и Аудио (<audio>)

  • Смысл: Плеер для медиа.
  • Закрывать: Да.
  • Где расположен: Внутри body.
  • Дочерние теги: <source>, <track>.
  • Атрибуты: src, controls (кнопки управления), autoplay, loop, muted.

11. Источник медиа (<source>)

  • Смысл: Указывает файл для video/audio/picture.
  • Закрывать: Нет (одинарный тег).
  • Где расположен: Внутри media-тегов.
  • Дочерние теги: Нет.
  • Атрибуты: src, type.

12. Векторная графика (<svg>)

  • Смысл: Контейнер для векторных рисунков.
  • Закрывать: Да.
  • Где расположен: Внутри body.
  • Дочерние теги: <circle>, <rect>, <path> и др.
  • Атрибуты: viewBox, width, height.

13. Адаптивное изображение (<picture>)

  • Смысл: Выбор разных картинок для разных экранов.
  • Закрывать: Да.
  • Где расположен: Внутри body.
  • Дочерние теги: Несколько <source> и один <img> в конце.
  • Атрибуты: Нет своих, атрибуты у дочерних source.

14. Гиперссылка (<a>)

  • Смысл: Ссылка на страницу, файл или место на странице.
  • Закрывать: Да (</a>).
  • Где расположен: В тексте (нельзя вкладывать ссылку в ссылку).
  • Дочерние теги: Текст, картинки, span и др.
  • Атрибуты: href (адрес, обязательно), target (_blank для новой вкладки), download, title.

15. Якорная ссылка

  • Смысл: Переход к элементу с определенным id на той же странице.
  • Как писать: <a href="#id_элемента">Текст</a>.
  • Нюансы: У элемента назначения должен быть атрибут id="..." совпадающий с href.
